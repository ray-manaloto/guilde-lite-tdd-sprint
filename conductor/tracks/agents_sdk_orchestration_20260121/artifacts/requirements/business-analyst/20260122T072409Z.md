# Requirements: CLI Interview Flow Inputs/Outputs

## Responsibility
- The CLI command responsible for the Conductor-first sprint interview (writes `conductor/tracks/<id>/spec.md` + `plan.md`) owns `--output-format` and `--output-schema`.
- This is distinct from the existing DB-only `spec-run` command.

## Inputs (minimum)
- `--track-id` or `--track-path` (select/create Conductor track).
- `--prompt` or `--prompt-file` (sprint prompt).
- `--max-questions` (default 5, per Ralph interview).
- `--output-format {md,json}` (stdout format).
- `--output-schema path/to/schema.json` (optional JSON schema override for stdout).

## Outputs
- Updates Conductor canonical artifacts:
  - `conductor/tracks/<id>/spec.md`
  - `conductor/tracks/<id>/plan.md`
- Writes interview Q/A capture into track artifacts for traceability.
- Prints a structured summary to stdout (format controlled by `--output-format`).

## Constraints
- No new PydanticAI usage in orchestration/planning; OpenAI Agents SDK + LiteLLM for Anthropic only.
- Do not remove PydanticAI dependencies until migration completes.
