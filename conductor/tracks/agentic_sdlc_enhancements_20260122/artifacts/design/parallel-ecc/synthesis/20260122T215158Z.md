   - Add `CODEX.md` (Codex equivalent of `CLAUDE.md`) as the canonical project context.
   - Place agents, skills, commands in your Codex tools/registry path (same layout as everything-claude-code).

2. **Codex lifecycle hooks**
   - Port `SessionStart/Stop/PreCompact` scripts into Codex wrapper scripts (or CI entrypoints) to maintain `codex-memory.md` and/or daily session logs.
   - If no `/compact`, append a manual summary into `codex-memory.md` when `strategic-compact` triggers.

3. **Workflow gates**
   - **PR Smoke (fast):** Prettier + TS check hooks, `/code-review`, `security-reviewer`, `/verify`, `/test-coverage`
   - **Nightly:** `/e2e` + `e2e-runner`, `/eval` with `eval-harness`, `build-error-resolver` for triage

4. **UI/UX adaptations**
   - Extend `code-reviewer` or create `ui-ux-reviewer` agent.
   - Add design references (Figma links, screenshots, tokens) to `CODEX.md`.

5. **Documentation**
   - Run `/update-docs` + `/update-codemaps` post‑implementation before PR creation.
   - Whitelist docs locations if using doc‑creation hooks.

## Do‑Not‑Adopt Items (and why)
- **PreToolUse hook that blocks new `.md/.txt` files** — will conflict with Codex doc workflows; too restrictive.
- **tmux reminder + dev‑server block hooks** — low value in Codex/CI environments.
- **git push pause hook** — interferes with automated CI/CD pipelines.
- **Auto‑merge of continuous‑learning outputs** — risks knowledge rot without review; keep manual approval.
- **Mandatory `/e2e` on every PR** — too slow; reserve for nightly or critical UI changes.
