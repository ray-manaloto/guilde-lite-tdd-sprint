## Recommended Adoption
1. **Adopt `skills/frontend-patterns/` + `/plan`** as the default UI implementation flow for Next.js work.  
   - Aligns with agentic SDLC best practice of structured, composable workflows.

2. **Adopt `/e2e` + `agents/e2e-runner.md`** for UI regression coverage (Playwright).  
   - Supports QA gates (smoke PR + nightly) in the research digest.

3. **Adopt `/code-review` + `agents/code-reviewer.md`**, but extend with a **UI/UX checklist** (contrast, spacing consistency, keyboard accessibility, responsive behavior).

4. **Adopt hooks:**
   - **Prettier auto-format** and **TypeScript check** for UI files.
   - **console.log warnings** for UI hygiene.
   - **memory-persistence + strategic-compact** for context recovery (aligns with context engineer role).

5. **Adopt `/verify` + `skills/verification-loop/`** to set up evaluator–optimizer loops for UI tasks (separate AI instance validates UI changes against requirements).

## Adaptation Notes
- **UI/UX-focused agent or skill**: create a new `ui-ux-reviewer.md` agent or extend `code-reviewer.md` with a UI/UX rubric (a11y, layout consistency, responsive behavior, visual regressions).
- **Design inputs**: add guidance in `CLAUDE.md` on referencing Figma links, screenshots, or design system tokens so UI work has explicit targets (per research digest).
- **FastAPI + Next.js**: include Next.js-specific conventions in `skills/frontend-patterns/` (App Router, server actions, data fetching patterns).
- **Documentation sync**: extend `doc-updater.md` to update UI docs (e.g., component library docs or storybook notes).

## Risks/Dependencies
- **Hook tooling**: Prettier/tsc hooks require these tools installed and configured in the repo.
- **False positives**: console.log warnings may block legitimate debug in early prototyping—define exception rules.
- **Over-automation risk**: UI changes may pass linting but fail UX goals; require explicit UX review or human checkpoint.
- **Context bloat**: memory persistence can accumulate stale UI requirements; needs curation.
